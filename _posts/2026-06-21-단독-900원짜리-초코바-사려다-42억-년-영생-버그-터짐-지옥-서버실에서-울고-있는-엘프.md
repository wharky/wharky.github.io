## 💡 도대체 뭐가 터진 건데? (초간단 비유)

일반인 분들을 위해 이번 장애의 핵심 범인 두 명을 아주 쉽게 비유해 드립니다.

### 1. 멱등성(Idempotency)의 가출
> **"배달 앱에서 '결제하기' 연타했다고 치킨이 50마리 배달되면 안 되잖아?"**

그게 바로 '멱등성'입니다. 아무리 버튼을 연타해도 결제는 딱 '한 번'만 되어야 하죠. 
근데 하데스-페이의 결제 API가 네트워크 에러를 만나자 정신을 놓았습니다. 김철수의 결제 요청을 다 다른 사람의 요청으로 인식해서 **수명 10분씩을 실시간으로 계속 깎아내린 겁니다.**

### 2. 정수 언더플로우(Integer Underflow)의 기적
> **"잔고가 0원인데 10원을 보냈더니, 갑자기 통장에 9999조 원이 찍혔다?"**

컴퓨터는 숫자를 저장하는 방이 정해져 있습니다. 
만약 수명을 저장하는 방이 `0`보다 작은 음수를 저장할 수 없는 '양수 전용 방(Unsigned Integer)'이라고 해봅시다.
`0`에서 `1`을 빼면 `-1`이 되어야 하죠? 하지만 컴퓨터는 뒤로 갈 수가 없어서, **시계바늘이 한 바퀴 돌아 가장 큰 숫자인 `4,294,967,295`로 워프**해 버립니다.

김철수 씨는 수명이 딱 5분 남은 상태였습니다. 
근데 재시도 공격으로 10분이 차감되자, 수명이 `-5분`이 된 게 아니라 **우주가 멸망해도 살아남을 '42억 년'**이 되어버린 겁니다.

## 💻 문제의 그 쓰레기 코드 (JS 가상 코드)

당시 결제 시스템의 백엔드 코드입니다. 기가 막힙니다.

```javascript
// 지옥 결제 대행 서비스 (하데스-Pay)
async function processSoulPayment(userId, priceMinutes) {
  // DB에서 유저의 남은 수명(분 단위, Unsigned Int)을 가져옴
  let currentLifespan = await db.getLifespan(userId); 

  console.log(`현재 고객 수명: ${currentLifespan}분`);

  // [치명적 버그 1] 멱등성 검증 없음. 요청 들어오는 대로 다 깎음.
  // [치명적 버그 2] 수명이 가격보다 적은지 검사 안 함. (안전장치 가출)
  let updatedLifespan = currentLifespan - priceMinutes; 

  if (updatedLifespan < 0) {
    // 이미 언더플로우가 터져서 updatedLifespan은 42억 년이 되었으므로, 
    // 이 if문은 영원히 실행되지 않습니다. (ㅋㅋ 망함)
    throw new Error("영혼 부족: 즉시 지옥행 버스 탑승 대상입니다.");
  }

  // DB에 저-장! (인간에서 신으로 진화하는 순간)
  await db.updateLifespan(userId, updatedLifespan);
  
  return "결제 완료! 오늘도 활기찬 하루 되세요!";
}
```
