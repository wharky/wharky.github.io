**문제: 2D 세상에 3D 환영을 입히는 법 (SNES Mode 7)**

SNES(슈퍼 패미컴)의 'Mode 7'은 레트로 그래픽의 정수라고 할 수 있습니다. 배경 레이어를 확대, 축소, 회전시켜 마치 3D 공간을 비행하거나 달리는 듯한 착각을 불러일으켰죠. F-Zero, 슈퍼 마리오 카트, 젤다의 전설: 신들의 트라이포스 등이 이 기술의 대표적인 예시입니다. 당시에는 혁명적인 기술이었지만, 본질적으로는 **하나의 2D 배경 이미지를 특정 행렬 연산을 통해 스크린에 투영하는 것**이었습니다. 하드웨어의 특정 레지스터를 조작하여 2D 이미지에 아핀 변환(affine transformation)을 적용한 것이었죠.

**해결: GLSL로 Mode 7의 마법을 재현하기**

이러한 Mode 7의 원리를 GLSL 셰이더로 재현하는 것은 아주 좋은 학습 자료이자 실제 게임에서도 유용한 레트로 스타일링 기법이 됩니다. 2D 평면 위에 3D 카메라가 위치한 것처럼 보이는 효과를 셰이더에서 직접 계산하는 거죠.

핵심 로직은 다음과 같습니다:
1.  **스크린 좌표를 가상의 3D 평면(월드 좌표)으로 변환**: 화면의 픽셀 위치를 텍스처가 그려질 가상의 바닥 평면 위 좌표로 매핑합니다.
2.  **카메라/시점 변환**: 이 평면 좌표에 '카메라'의 위치, 회전, 확대/축소 등 변환 행렬을 역순으로 적용합니다. (즉, 카메라 움직임을 평면 자체에 적용하는 것처럼)
3.  **텍스처 샘플링**: 변환된 좌표를 이용해 원본 배경 텍스처에서 색상을 가져옵니다.

이를 의사코드(Pseudo-code)로 살펴보면:

```glsl
// Fragment Shader Logic for Mode 7 like effect
// uniform vec2 iResolution;    // Screen resolution (예: vec2(1920, 1080))
// uniform float iTime;        // Shader 시간 (예: 0.0부터 증가)
// uniform sampler2D iChannel0; // Mode 7 배경 텍스처 (예: 도로, 바닥 이미지)

void mainImage(out vec4 fragColor, in vec2 fragCoord) {
    // 1. 화면 픽셀 좌표를 정규화된 공간 [-1, 1]로 변환 후, 종횡비 보정
    vec2 uv = (fragCoord / iResolution) * 2.0 - 1.0; 
    uv.y *= -1.0; // Y축 뒤집기 (GLSL은 보통 아래가 -1, 위가 1)
    uv.x *= iResolution.x / iResolution.y; // 종횡비 보정

    // 가상 카메라 설정
    // 시간이 지남에 따라 카메라가 회전하고 움직이는 효과를 줍니다.
    float cameraZoom = 0.5 + abs(sin(iTime * 0.3)) * 0.4; // 시간에 따른 줌 효과
    vec2 cameraOffset = vec2(sin(iTime * 0.2) * 2.0, cos(iTime * 0.3) * 2.0 + iTime * 0.5); // 카메라 이동
    float cameraAngle = iTime * 0.5; // 시간에 따른 회전 효과 (라디안)

    // '바닥' 평면의 가상 월드 좌표 계산 (카메라가 평면 위를 이동하는 것처럼)
    // 중요한 점은, 화면 픽셀(uv)을 월드 공간의 점으로 변환해야 한다는 것입니다.
    // 여기서는 간단하게, 카메라 위치와 회전을 '역으로' uv에 적용합니다.
    
    // 줌 적용 (거리 개념)
    vec2 worldPoint = uv / cameraZoom;

    // 회전 적용
    float cosA = cos(cameraAngle);
    float sinA = sin(cameraAngle);
    mat2 rotMatrix = mat2(cosA, -sinA, sinA, cosA);
    worldPoint = rotMatrix * worldPoint; // 화면 중앙을 기준으로 회전

    // 카메라 이동 적용 (월드 좌표로 변환된 후의 이동)
    worldPoint += cameraOffset;

    // 2. 텍스처 샘플링 (원본 텍스처는 바둑판 무늬나 길 패턴이면 효과적)
    // worldPoint를 다시 [0,1] 범위의 텍스처 UV로 사용. mod를 사용하여 텍스처 반복 효과
    // Mode 7은 텍스처가 반복되는 경우가 많으므로 mod를 사용합니다.
    vec2 texUV = mod(worldPoint, 1.0); 
    texUV = fract(worldPoint); // 또는 fract() 함수로 소수점 부분만 사용
    if (texUV.x < 0.0) texUV.x += 1.0; // fract는 음수도 반환하므로 0~1로 맞춰줍니다.
    if (texUV.y < 0.0) texUV.y += 1.0;

    // 3. 최종 색상 계산
    fragColor = texture(iChannel0, texUV);
}
```

이 코드는 단순화된 예시지만, `uv` 좌표에 카메라 이동(translation), 회전(rotation), 스케일(scale) 변환을 가하고, `mod` 또는 `fract` 함수로 텍스처를 반복시키는 과정을 통해 '가짜 3D' 효과를 만들어냅니다. `mat2`를 이용한 회전이나 `mod`를 이용한 반복 패턴은 GLSL 셰이더 수학의 기본이자 핵심입니다. 닌텐도 개발자들이 어셈블리어로 한땀 한땀 구현했던 로직을, 우리는 GLSL의 강력한 벡터 및 행렬 연산을 활용해 훨씬 직관적으로 구현할 수 있는 거죠.

**문제: 물결 효과, 연기, 흔들림 (N64 젤다의 전설: 시간의 오카리나 물결)**

N64의 '젤다의 전설: 시간의 오카리나'를 플레이해본 분들이라면, 물 표면의 잔잔한 움직임이나 특정 환경 효과들이 아직도 기억에 남을 겁니다. 물론 지금 보면 투박하지만, 그 당시에는 정말 생생했죠. 이는 픽셀의 위치를 직접적으로 조작하거나, 텍스처 좌표를 미세하게 흔들어주는 방식으로 구현되었습니다.

**해결: GLSL 사인 함수로 파동을 만들다**

현대 셰이더에서는 `sin`과 `cos` 같은 삼각함수를 이용해 이런 파동 효과를 아주 쉽게 만들 수 있습니다. 텍스처를 샘플링할 때 사용되는 UV 좌표를 시간과 공간에 따라 변형시키는 것이죠.

```glsl
// Fragment Shader Logic for Water/Distortion effect
// uniform vec2 iResolution;    // Screen resolution
// uniform float iTime;        // Shader time
// uniform sampler2D iChannel0; // Base texture (예: 물 표면, 배경 이미지)

void mainImage(out vec4 fragColor, in vec2 fragCoord) {
    vec2 uv = fragCoord / iResolution; // 0.0 ~ 1.0 범위의 정규화된 UV

    // 파동 매개변수
    float waveStrength = 0.02;     // 파동의 최대 강도
    float waveSpeed = 3.0;         // 파동의 속도
    float waveDensity = 30.0;      // 파동의 밀도 (얼마나 촘촘하게 파동이 생기는지)

    // UV 좌표를 사인 함수로 왜곡
    // x축 방향으로 y값에 따라 파동을 적용합니다.
    // iTime을 더해 시간이 지남에 따라 파동이 흐르게 합니다.
    uv.x += sin(uv.y * waveDensity + iTime * waveSpeed) * waveStrength;
    
    // y축 방향으로도 파동을 적용하여 더욱 자연스러운 물결을 만듭니다.
    uv.y += cos(uv.x * waveDensity * 0.7 + iTime * waveSpeed * 1.2) * waveStrength * 0.8;

    // (선택) 경계 처리: 왜곡된 UV가 0~1 범위를 벗어날 경우, Clamp 또는 반복 처리
    uv = clamp(uv, 0.0, 1.0); // 텍스처 밖으로 나가지 않도록 고정

    fragColor = texture(iChannel0, uv);
}
```

이 코드에서 `sin` 함수는 `uv.y` (세로축)에 따라 `uv.x` (가로축)를 앞뒤로 흔들리게 만듭니다. `iTime`을 곱해주면 시간이 지남에 따라 파동이 움직이는 애니메이션 효과가 생기죠. `waveDensity`와 `waveStrength`는 파동의 모양을 조절합니다. 이런 식으로 픽셀의 텍스처 좌표를 수학적으로 조작함으로써, 우리는 실제처럼 움직이는 물결이나 일렁이는 열기, 흔들리는 아지랑이 같은 효과를 만들어낼 수 있습니다. 닌텐도 개발자들이 한정된 리소스로 수없이 많은 시행착오를 거쳐 찾아냈을 '최적의 파동 공식'을, 우리는 몇 줄의 GLSL 코드로 쉽게 실험하고 구현할 수 있는 겁니다.

**선배의 썰: 제한이 창의력을 낳는다는 착각과 현실**

저도 현업에서 수많은 프로젝트를 거쳐왔지만, 가끔 레트로 게임을 파고들다 보면 뼈저리게 느끼는 것이 있습니다. "제한이 창의력을 낳는다"는 말은 반은 맞고 반은 틀립니다. 제한된 환경에서 대단한 결과물을 뽑아낸 것은 맞지만, 개발자들이 그 과정에서 겪었을 고통과 좌절은 상상 이상이었을 겁니다. 지금은 풍부한 하드웨어 리소스와 강력한 셰이더 언어가 우리에게 주어져 있습니다. 하지만 그렇다고 해서 우리가 무조건 더 좋은 결과를 낸다고는 할 수 없습니다.

오히려 레트로 개발자들의 '최적화 정신'과 '수학적 통찰'은 지금 우리에게 더 큰 영감을 줍니다. 그들이 온전히 수학적인 계산과 리소스 관리만으로 이룩했던 효과들을 보면서, 우리는 GLSL이라는 강력한 도구를 어떻게 하면 더욱 효율적이고 예술적으로 활용할 수 있을지 고민하게 됩니다. 단순한 텍스처 한 장으로 구현할 수 있는 것을 거대한 모델링으로 해결하려 하거나, 불필요한 연산을 덕지덕지 붙이는 대신, 픽셀 하나하나의 의미를 되새기며 수학적인 우아함으로 문제를 해결하는 방법을 배울 수 있다는 거죠. 이건 마치 '더 좋은 도끼를 가졌으니 더 큰 나무만 벨 거야!'가 아니라, '더 좋은 도끼로 어떻게 하면 가장 효율적이고 아름다운 조각품을 만들까?'를 고민하는 것과 같습니다.
