## 🛠️ 눈물겨운 솔루션 아키텍처: 뭐가 터진 걸까?

이 장애를 분석하려고 코드를 열어본 순간, 난 내 송곳니를 스스로 뽑아버리고 싶었다.

쉽게 비유해 줄게.
원래 결제 시스템은 마트 계산대랑 똑같아. 
물건(기적)을 가져오면 계산원(서버)이 바코드를 찍고, 네 지갑(영혼)에서 돈(숙면 시간)을 빼가야 한다.

그런데 신입 엘프 개발자 녀석이 **'자바스크립트의 고질적인 소수점 연산 오류'**를 간과한 거다.

### 1. 0.1 + 0.2는 0.3이 아니다?
자바스크립트 이 멍청한 녀석은 10진수 소수점을 2진수로 바꾸느라 `0.1 + 0.2`를 계산하면 `0.30000000000000004` 같은 기괴한 숫자를 뱉는다. 
너희가 마트에서 300원짜리 사탕을 샀는데, 계산원이 **"손님, 0.0000000000000004원이 부족해서 결제가 안 됩니다"**라며 네 영혼을 압류해 버린 상황인 거지.

### 2. 'NaN(Not a Number)'의 저주
할머니가 적용한 할인 쿠폰이 만료된 쿠폰이었다. 
시스템은 존재하지 않는 값(`undefined`)에 숙면 주파수를 곱해 버렸고, 그 결과 결제 금액이 **`NaN` (숫자가 아님)**이 되었다.

이건 쉽게 말해, 계산원이 **"손님, 결제 금액이 '바나나' 원입니다. 지갑에서 '바나나'만큼의 영혼을 꺼내주세요"**라고 요구한 거다. 
컴퓨터는 '바나나'가 뭔지 몰라 뇌정지가 왔고, 결국 결제 대상자인 할머니의 영혼 데이터 자체를 포맷해 버렸다.

## 💻 문제의 코드와 눈물의 심폐소생술

그때 내가 급하게 작성했던 긴급 패치 코드다. 
주석을 잘 읽어보면 눈물 없이는 볼 수 없을 거다.

```javascript
// =================================================================
// [긴급 패치] 영혼 결제 예외 처리 (작성자: 가르쉬 시니어 개발자)
// 코드를 건드리면 다음 생에 웰시코기로 환생할 줄 알아라.
// =================================================================

function processSoulPayment(userId, cartItem, paymentSource) {
  // 1. 결제 금액 계산 (할인율 적용)
  let rawPrice = cartItem.basePrice * (1 - paymentSource.couponDiscount);
  
  // [기존 버그]: rawPrice가 0.3000000000004 가 됨
  // [해결 비유]: 0.1원 단위 쪼개기 방지용 무쇠 가마솥 필터 장착!
  let exactPrice = Math.round(rawPrice * 100) / 100;

  // 2. NaN (바나나 현상) 원천 차단
  if (isNaN(exactPrice) || exactPrice <= 0) {
    // [비유]: 결제 금액이 미쳐 날뛰면 거래를 즉시 취소하고,
    // 대악마 바알의 잔소리 스크립트를 출력한다.
    throw new Error("[CRITICAL] 결제 금액이 숫자가 아닙니다! 영혼 압류를 중단합니다.");
  }

  // 3. 영혼 원장에서 차감 시도
  const userSoul = database.findSoulById(userId);
  
  if (userSoul.residualEnergy < exactPrice) {
    return "잔액 부족: 다음 주 월요일의 피로도를 더 판매하십시오.";
  }

  // 안전장치: 영혼의 핵심 코어(Root Core)가 완전히 지워지는 것 방지
  if (exactPrice >= userSoul.totalVitality) {
    console.warn(`[경고] 유저 ${userId}가 자살 기도를 결제로 위장함. 강제 반려.`);
    return "거래 거부: 목숨값은 결제 수단으로 사용할 수 없습니다.";
  }

  // 실제 영혼 차감 연산
  userSoul.residualEnergy -= exactPrice;
  return "결제 완료: 기적이 배송 중입니다.";
}
```
