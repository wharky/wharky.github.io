## 야근 중에 날아온 우주적 대재앙 (ITSM Ticket #9999)

여느 때와 다름없는 목요일 밤이었습니다. 뜨끈한 디카페인 마나 포션을 마시며 퇴근 랙을 만끽하려던 찰나, 모니터가 시뻘건 경고등으로 도배되기 시작했습니다. 

사내 메신저 슬랙(Slack)이 비명을 지릅니다. 알람음이 너무 울려서 맥북이 공중 부양할 기세더군요. 정신을 차리고 티켓 시스템(ITSM)을 열어봤습니다. 

> **[우주 인프라 관리국 - 긴급 장애 티켓]**
> * **등급**: **P0 (우주 소멸 등급)**
> * **대상 장비**: 멀티버스 게이트웨이 라우터 (MGR-8008)
> * **장애 요약**: 지구-302 구역에서 출발한 피자 배달원이 포탈 진입 후 무한 루프에 빠짐.
> * **현 상황**: 
>   1. 배달원이 포탈을 통과하는 순간 **500 Internal Server Error** 발생.
>   2. 포탈 리다이렉트(Redirect) 주소가 자기 자신으로 잘못 세팅됨.
>   3. 배달원이 0.0001초마다 똑같은 포탈 입구로 계속 튕겨져 나오는 중.
>   4. 현재 Kepler-186f 행성 출구 쪽에 **450만 명의 피자 배달원**이 물리적으로 복사되어 쌓여 있음. 행성 중력 붕괴 직전.

쉽게 말해 이겁니다. 

여러분, 노래방에서 마이크를 스피커에 가까이 대면 **"삐이이이-"** 하고 찢어지는 소리 나죠? 소리가 스피커로 나갔다가 다시 마이크로 들어가서 무한히 증폭되는 현상, 그걸 **하울링(Howling)**이라고 합니다.

지금 그 하울링이 **'사람의 육체'**와 **'배달용 페퍼로니 피자'**로 일어나고 있는 겁니다. 공간 차원 서버가 맛이 가면서, 나갔던 배달원이 다시 입구로 들어오는 무한 루프 버그가 터진 것이죠.

## 뇌 정지 온 시니어 개발자의 눈물겨운 대처법

"그냥 서버 전원 플러그 뽑으면 안 돼요?"

이런 소리 하는 신입 기획자는 차원 밖으로 던져버리고 싶습니다. 이 라우터 서버는 우주 전체의 공간을 연결하는 '엄마 서버'입니다. **서버를 강제 종료(Reboot)하는 순간, 현재 차원 터널을 통과하고 있던 70억 명의 여행객들이 우주 먼지로 포맷됩니다.** 제 KPI와 목숨이 동시에 날아가는 시나리오죠.

살아있는 상태로 수술을 해야 합니다. 일명 **'라이브 핫패치(Live Hot-patch)'**.

우선 이 미쳐버린 무한 루프의 원인을 파악했습니다.

1. **상황 분석**: 
   포탈을 통과할 때마다 사용자의 위치 데이터(`X, Y, Z, Dimension_ID`)를 업데이트해야 합니다. 그런데 우주 패치 후 데이터베이스가 뻗었습니다. 
2. **버그의 탄생**: 
   서버가 "야, 얘 어디로 보내야 해?"라고 물었는데 DB가 응답이 없으니, 기본값(Default)인 `Current_Location`으로 목적지를 잡아버린 겁니다. 즉, "너 있는 곳으로 다시 가!"가 된 거죠.
3. **지옥의 굴레**: 
   배달원은 1초에 수천 번씩 '나'에게서 출발해 '나'에게로 도착하며 무한 복사되고 있었습니다.

우리가 취한 해결책은 **'쓰레기 수거차(Garbage Collector)'**를 코드에 강제로 심는 것이었습니다. 루프가 돌 때마다 똑같은 녀석인지 검사하고, "어? 아까 본 배달원이네?" 싶으면 즉시 차원 밖 외딴 쓰레기통 차원(Earth-404, 아무것도 없는 무의 공간)으로 튕겨내도록 임시 방어벽을 세우는 코드를 작성했습니다.

## 문제의 버그 코드와 눈물의 심폐소생술 (JS 가상 코드)

당시 서버의 심장을 부여잡고 급하게 밀어 넣었던 자바스크립트 패치 코드입니다. 읽어보면 왜 터졌는지 바로 이해가 가실 겁니다.

```javascript
// [주의] 섣불리 건드리면 우주가 압축 파일(.zip)이 될 수 있습니다.
async function portalRoutingSystem(traveller) {
  try {
    // 1. 목적지 차원의 IP 주소를 가져온다.
    let destination = await getDimensionIP(traveller.targetDimension);
    
    // [버그 터진 구간] DB가 응답 없으면 destination이 undefined가 됨!
    if (!destination) {
      console.error("⚠️ 차원 DB 사망! 긴급 우회 주소를 세팅합니다.");
      
      // 원래는 에러 페이지로 보내야 하는데, 신입이 코드를 이따위로 짰음.
      // "에러 났어? 그럼 그냥 출발했던 포탈 입구로 다시 돌려보내!" -> 지옥의 시작
      destination = traveller.currentPortalEntrance; 
    }

    // 2. 물리적 차원 전송 실행 (여기가 무한 루프 지옥)
    await teleportPhysicalBody(traveller, destination);

  } catch (cosmicError) {
    // [긴급 핫패치] 무한 루프 감지 및 강제 탈출 장치 가동!
    if (traveller.loopCount > 100) {
      console.warn(`🚨 경보: 동일 개체 무한 루프 감지! ID: ${traveller.id}`);
      
      // 배달원을 아무것도 없는 '404 임시 대기실'로 리다이렉트
      let trashDimension = "Dimension-404-VOID"; 
      
      // 복사된 450만 명의 배달원을 한 곳으로 모아 안전하게 병합(Merge) 처리
      await mergeDuplicatedAtoms(traveller);
      
      return redirect(traveller, trashDimension);
    }
    
    traveller.loopCount++;
    throw cosmicError;
  }
}
```
