## 2. 눈물겨운 솔루션 아키텍처: '차원 쓰레기 수거차' 가동

이미 물리적인 서버실은 슬라임 신격으로 가득 차서 진입이 불가능했습니다. 원격으로 해결해야 했습니다.

이 소환 자동화 카탈로그의 가장 큰 문제는 **'가비지 컬렉터(Garbage Collector)'**가 없었다는 겁니다. 

*   **가비지 컬렉터가 뭔데?**
    *   쉽게 말해 **'자취방 청소해 주는 엄마'**입니다. 여러분이 물을 마시고 컵을 내버려 두면(메모리 낭비), 엄마가 스윽 와서 설거지통에 넣죠(메모리 회수).
    *   그런데 이 미친 교단 놈들은 소환 코드만 짜놓고, 소환된 신격을 회수하는 '퇴마(Release)' 코드를 안 짰습니다. 먹튀한 거죠.

저는 이 문제를 해결하기 위해 **[시공간 프록시 우회 및 강제 가비지 컬렉션 아키텍처]**를 설계했습니다.

1.  **로드 밸런서(IP 분배기) 차단**: 소환 신호가 서버실로 직접 들어오지 못하도록, 차원 관문을 블랙홀(Null) 계정으로 강제 포워딩했습니다. (오는 대로 다 쓰레기통에 처박기)
2.  **타임 아웃 오버라이드**: 소환 요청이 0.1초 이상 반응이 없으면 자동으로 '소환 계약 파기'가 되도록 우주의 물리 법칙(Config)을 강제로 수정했습니다.

## 3. 범인 검거: 범죄 현장의 JavaScript 코드

교단 인턴 놈이 짠 문제의 코드입니다. 자바스크립트를 조금이라도 아시는 분은 기겁할 겁니다. 몰라도 주석을 보면 얼마나 미친 짓인지 알 수 있습니다.

```javascript
// 고대신 카탈로그 장바구니 결제 모듈 (v1.0.4-buggy)
async function processSubscription(userId, deityId) {
  
  // 1. 소환 루프 시작 (근데 멈추는 조건문이 없음)
  while (true) { 
    try {
      const portal = await openDimensionalPortal('Void-9');
      const summonDeity = await portal.grabDeity(deityId);
      
      // 2. 소환된 신격을 서버실 메모리(물리 공간)에 강제 렌더링
      serverRoom.physicalSpace.push(summonDeity); 
      
      console.log(`[성공] ${deityId}가 소환되었습니다. 신도들은 절하십시오.`);
      
      // 3. 원래는 30일에 한 번 소환해야 하는데...
      // 개발자가 '밀리초(ms)' 단위를 헷갈려서 30일을 30ms로 설정함.
      // 즉, 0.03초마다 한 번씩 소환 중.
      await sleep(30); 

    } catch (error) {
      // 4. 에러가 나면 멈춰야 하는데, "신앙심 부족"이라며 무한 재시도 함.
      console.error("차원 렉 발생! 더 강하게 소환합니다!");
      continue; 
    }
  }
}

// 개발자의 유언: "어차피 우주 멸망할 건데 메모리 누수 좀 나면 어때?"
```
