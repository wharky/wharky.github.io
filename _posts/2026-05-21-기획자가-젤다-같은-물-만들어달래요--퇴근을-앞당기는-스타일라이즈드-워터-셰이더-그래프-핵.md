## 1. 씬 뎁스(Scene Depth)로 해안가 거품(Foam) 날로 먹기

스타일라이즈드 물의 생명은 물과 땅이 만나는 지점에 생기는 **'하얀 거품(Shoreline Foam)'**입니다. 이걸 메시로 일일이 배치한다? 당장 그만두세요. 우리는 셰이더 안에서 카메라와 지형의 거리를 계산해 자동으로 거품을 그려줄 겁니다.

원리는 간단합니다. **'현재 물 표면의 깊이'**와 **'그 뒤에 있는 지형의 실제 깊이'**의 차이를 계산하는 것이죠.

### 핵심 메커니즘: Scene Depth - Screen Position.w
*   **Scene Depth**: 카메라로부터 물 뒤에 있는 불투명 객체(땅, 바위 등)까지의 거리.
*   **Screen Position.w (혹은 Pixel Depth)**: 카메라로부터 현재 물 표면 픽셀까지의 거리.

이 둘을 빼면 어떻게 될까요? 물이 땅과 가까워질수록(얕아질수록) 두 값의 차이는 `0`에 수렴합니다. 이 영역을 하얗게 칠해주면 그게 바로 거품이 됩니다.

```hlsl
// [Pseudo-code] 해안가 거품 영역 마스크 계산
float RawDepth = SampleSceneDepth(screenUV);
float LinearDepth = LinearEyeDepth(RawDepth); // 플랫폼별 깊이 비선형성 보정
float WaterSurfaceDepth = screenPos.w;

// 물의 깊이 차이 계산
float DepthDifference = LinearDepth - WaterSurfaceDepth;

// 거품의 범위를 조절하는 파라미터(FoamWidth)로 나누고 클램핑
float FoamMask = 1.0 - saturate(DepthDifference / _FoamWidth);
```

여기에 단순히 일정한 띠만 생기면 심심하겠죠? `FoamMask`에 **시간(Time)에 따라 흐르는 노이즈 텍스처(Gradient Noise)**를 곱해줍니다. 그러면 거품이 파도 치듯 구불구불 움직이는 다이내믹한 연출이 완성됩니다.

## 2. 노멀 맵(Normal Map) 듀얼 스크롤링: 반복 패턴의 저주 풀기

물이 흐르는 느낌을 주려면 노멀 맵이나 보로노이(Voronoi) 노이즈를 움직여야(Panning) 합니다. 하지만 한 방향으로만 주구장창 흘려보내면 플레이어는 3초 만에 눈치챕니다. *"어, 이거 텍스처 흘려보내는 거네?"*

이 반복 패턴의 저주를 깨는 기가 막힌 가성비 기술이 바로 **'듀얼 스크롤링(Dual Scrolling)'**입니다.

### 구현 알고리즘 구조

방법은 간단하지만 강력합니다. 크기가 다르고 흐르는 방향이 미세하게 다른 두 개의 노멀 맵(또는 보로노이 노이즈)을 섞는(Blend) 것입니다.

```
[Normal Map A] -> Tiling (2.0, 2.0) -> Scroll (Speed: X=0.1, Y=0.05)   \\
                                                                        ---> [Blend Normal Node] -> Final Normal
[Normal Map B] -> Tiling (1.3, 1.3) -> Scroll (Speed: X=-0.08, Y=-0.07) /
```

*   **스케일 비대칭**: 하나는 크게(`Tiling 1.0`), 하나는 작게(`Tiling 1.7`) 세팅합니다.
*   **방향 교차**: 하나는 우상향, 하나는 좌하향으로 흐르게 만듭니다. 소수점 단위로 속도를 꼬아놓아야 패턴이 겹치는 주기가 길어집니다.
*   **블렌딩**: 단순히 더하는(`+`) 것이 아니라, 셰이더 그래프의 **'Normal Blend'** 노드(RNM: Reoriented Normal Mapping 방식)를 사용해야 셰이딩이 깨지지 않고 자연스럽게 합성됩니다.

이렇게 섞인 노멀 값을 물 표면의 굴절(Refraction)과 스펙큘러(Specular) 계산에 입력하면, 정말 살아있는 듯한 유기적인 물결이 연출됩니다.

## 3. 리프랙션(Refraction) 디스토션: 픽셀을 쥐어짜는 마술

스타일라이즈드 물 안쪽이 투명하게 비치되, 물결 때문에 굴절되어 흔들려 보이면 퀄리티가 200% 상승합니다. 

이걸 매번 진짜 굴절 레이트레이싱을 돌릴 순 없으니, 우리는 **화면 캡처 텍스처(Scene Color / Camera Opaque Texture)**를 가져와서 장난질을 칠 겁니다.

### 굴절 왜곡 알고리즘
물 표면을 그릴 때, 물 뒤에 렌더링된 화면 이미지를 가져와 스크린 좌표에 약간의 오프셋(Offset)을 줘서 왜곡시킵니다. 이 오프셋의 기준이 바로 앞서 만든 **듀얼 스크롤링 노멀 맵의 R, G 채널 값**입니다.

```hlsl
// [Pseudo-code] 굴절 효과 구현
float2 ScreenUV = ScreenPosition.xy / ScreenPosition.w;

// 노멀 맵의 XY(R,G) 채널을 왜곡 강도(_RefractionStrength)와 곱해 오프셋 생성
float2 DistortionOffset = WaterNormal.xy * _RefractionStrength;

// 깊이가 너무 얕은 곳(해안가 아주 가까이)에서는 왜곡이 튀지 않도록 DepthDifference로 감쇄
DistortionOffset *= saturate(DepthDifference * _DistortionFade);

// 최종 왜곡된 UV로 씬 컬러 텍스처 샘플링
float3 RefractedColor = SampleCameraOpaqueTexture(ScreenUV + DistortionOffset);
```

이때 꿀팁 하나. **지형 경계선 부분의 아티팩트 방지**입니다. 
굴절 값을 너무 세게 주면 물 위에 튀어나온 바위나 캐릭터가 물속 화면에 들어가서 왜곡되는 괴현상(Bleeding)이 발생합니다. 이를 막기 위해 위 코드처럼 `DepthDifference` 값을 왜곡 강도에 곱해 물이 얕은 곳에서는 왜곡이 자연스럽게 `0`으로 수렴하도록 제어해 줘야 합니다. 이게 바로 주니어와 시니어의 디테일 차이죠.

## 4. 최종 아키텍처 한눈에 보기 (Mental Map)

이 모든 요소를 결합한 스타일라이즈드 워터 셰이더의 전체 흐름도입니다. 머릿속으로 셰이더 그래프를 그려보세요.

```
1. Vertex Shader -> 간단한 Sine 웨이브로 정점 높이 살짝 흔들기 (선택 사항)
2. Fragment Shader
    ├── [Color Pass] Deep Water Color와 Shallow Water Color를 Scene Depth 기준으로 Lerp
    ├── [Foam Pass] (Scene Depth - ScreenPos.w) + Voronoi Noise -> 뽀글거리는 Shoreline 생성
    ├── [Normal Pass] Dual Scrolling Voronoi/Normal -> 물결 모양 제어
    └── [Refraction Pass] Normal 데이터를 이용해 Scene Color UV를 왜곡하여 합성
```
