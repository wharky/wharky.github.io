## 💡 원인 분석: 범인은 'display: none'과 무한 스크롤의 끔찍한 콜라보

문제는 그들이 구현한 **'무한 스크롤(Infinite Scroll)'**에 있었다.

보통 인스타그램이나 페이스북을 할 때, 화면을 아래로 내리면 새로운 피드가 계속 로딩된다. 이게 무한 스크롤이다. 작동 원리는 단순하다. 
*"사용자의 화면 눈높이가 웹페이지 맨 밑바닥에 닿았는가? -> 그렇다면 다음 10개를 더 불러와라."*

그런데 이 초짜 종교 개발자들이 치명적인 실수를 저질렀다.

카탈로그 화면을 예쁘게 보이게 하겠다고, 아직 불러오지 않은 고대신 목록 카드의 CSS 스타일을 `display: none`으로 숨겨놓은 것이다!

### 찰떡 비유로 이해하는 장애 메커니즘
이게 얼마나 멍청한 짓인지 아주 쉽게 설명해주겠다.

1. 당신이 무한 리필 회전초밥집에 갔다.
2. 당신 앞의 접시가 비면(눈높이가 바닥에 닿으면) 주방장은 즉시 새 초밥을 만들어 올린다.
3. 그런데 당신이 **투명 망토**를 쓰고 초밥을 먹고 있다.
4. 주방장 시점에서는? **"어? 손님 앞에 접시가 없네? (사실 투명해서 안 보임) 초밥 더 줘!!!"**
5. 주방장이 초광속으로 초밥을 1초에 50만 개씩 접시에 던지기 시작한다.
6. 결국 가게는 초밥 더미에 묻혀 폭발한다.

코드가 딱 이 꼴이었다. 
브라우저는 카탈로그 카드가 숨겨져 있으니까 **"어? 아직 바닥에 안 닿았네? 다음 고대신 불러와!"**를 무한대로 호출했다. 

그 결과, 0.1초 만에 API 서버가 심연의 차원에 **"고대신 4,200,000마리 일시불 결제요!"**라는 신호를 보내버린 것이다.

## 🛠️ 눈물겨운 솔루션 아키텍처: 촉수 분쇄기 구축하기

이미 소환 게이트는 열렸고, 촉수들은 랜선을 타고 랜섬웨어마냥 뻗어 나오고 있었다. 서버를 그냥 꺼버리면 그 반동으로 지구의 절반이 디멘션 시프트(차원 이동)될 판이었다. 

나는 세 가지 단계로 응급 수술에 들어갔다.

### 1. 차원 디바운싱 (Debounce) 도입
주방장이 미쳐 날뛰지 못하도록 브레이크를 걸어야 했다. 스크롤 이벤트가 일어날 때마다 소환 요청을 보내는 게 아니라, 스크롤이 완전히 멈추고 최소 0.3초는 지나야 딱 한 번만 소환 요청을 보내도록 제한했다.

### 2. 가상 스크롤 (Virtual Scroll) 성수 필터링
화면에 보이는 5마리의 고대신만 실제로 렌더링하고, 화면 밖으로 벗어난 고대신들은 즉시 메모리에서 지워버리는(DOM에서 제거) **가상 스크롤**을 적용했다. 아무리 스크롤을 내려도 우리 서버실 메모리에는 딱 5마리의 데이터만 남게 된다.

### 3. 강제 가비지 컬렉션 (Exorcism)
이미 불러와서 서버실 구석에 껴있는 400만 마리의 하급 몬스터 잔해들을 메모리에서 완전히 소멸시키기 위해 강제로 가비지 컬렉터를 구동시켰다. (성수를 서버 랙에 뿌리는 것과 비슷한 효과다.)

## 💻 문제의 그 무시무시한 JavaScript 가상 코드

당시 내가 덜덜 떠는 손으로 타이핑했던 그리스어 스크립트 기반의 자바스크립트 코드다. 주석을 잘 읽어봐라. 얼마나 절박했는지 보일 것이다.

```javascript
// ⚠️ 경고: 이 함수를 동기식(Sync)으로 호출하면 세상이 종말합니다.
async function summonCatalogInfiniteScroll() {
  const container = document.getElementById('abyss-list');
  
  // [버그 지점] container가 숨겨져 있어서 height가 0이 됨.
  // 브라우저는 끝없이 "더 불러와!"를 외치는 중.
  while (container.clientHeight === 0) {
    console.warn("🚨 [경고] 차원 높이가 0입니다! 무한 소환 루프 감지!");
    
    // 긴급 핫픽스: 디바운스(Debounce) 물리적 적용
    // 0.5초 동안 대기하며 심연의 API가 숨 쉴 틈을 준다.
    await new Promise(resolve => setTimeout(resolve, 500));
    
    const ancientGods = await fetchNextDeitiesFromServer(10);
    
    if (ancientGods.length === 0) break;

    ancientGods.forEach(god => {
      // 투명화 버그 해결을 위해 display: none 강제 해제
      const card = document.createElement('div');
      card.style.display = 'block'; 
      card.style.color = 'purple';
      card.innerText = `[소환됨] 이름: ${god.name} (위험도: 💀💀💀)`;
      
      // 장바구니에 담기자마자 날뛰는 것 방지 (Event Prevent)
      card.addEventListener('click', (e) => {
        e.preventDefault();
        alert("이미 영혼이 저당 잡힌 소환수입니다.");
      });

      container.appendChild(card);
    });

    // 가상 스크롤 가동: 화면에 10개 초과하면 맨 위 노드부터 가차없이 소멸
    while (container.children.length > 10) {
      console.log("🧹 가비지 컬렉터 가동: 안 보이는 고대신 환불 처리 중...");
      container.removeChild(container.firstChild); 
    }
  }
}
```

## 🤦\u200d♂️ 개발자의 넋두리: 제발 소환 의식에 Node.js 쓰지 마세요

장애 대응 시작 4시간 만에 상황은 진정되었다. 

무한 루프가 멈추자 서버실을 가득 채웠던 보라색 안개와 촉수들은 마치 브라우저 탭을 닫았을 때처럼 허무하게 증발했다. 고대어로 노래를 부르던 인턴도 정신을 차렸다. (자기가 쓴 자바스크립트 코드를 보고 괴성을 지르는 걸 보니 완벽히 정상으로 돌아온 게 맞다.)

내가 이 업계 후배들에게 뼈저리게 당부하고 싶은 말이 있다.

제발 고대 신을 소환하든, 우주의 비밀을 파헤치든, **기본적인 웹 표준과 CSS 박스 모델부터 공부해라.** 

`display: none`과 `Infinite Scroll`을 대책 없이 섞어 쓰면, 당신의 서버실은 진짜 지옥의 게이트가 될 수 있다. 그리고 이단 종교 교주님들아, 소환 의식 백엔드를 Node.js로 짜지 마라. 싱글 스레드라서 고대신 한 마리 광역 공격 캐스팅 들어가면 서버 전체가 멈춘다. 다음부턴 무조건 Go 언어로 고성능 멀티스레딩 소환 하길 바란다. 

나? 나는 선금 받은 거 챙겨서 이 우주에서 로그아웃한다. 다음 차원 고객사 미팅이 있어서 이만!
