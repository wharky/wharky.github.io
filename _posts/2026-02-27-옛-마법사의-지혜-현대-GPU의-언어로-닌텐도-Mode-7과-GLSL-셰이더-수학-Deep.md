### 문제: 과거의 마법을 재현하고 싶은데, 어떻게?

솔직히 고백하자면, 저처럼 나이 지긋한 개발자들은 가끔 이런 생각을 합니다. "요즘 게임들, 그래픽은 끝내주지만, 그 시절 닌텐도 게임에서 느꼈던 독특한 '감성'은 어디 갔을까?" 물론, 단순히 도트 그래픽을 박아 넣는다고 해결될 문제는 아닙니다. 진짜 핵심은, 그 시절 개발자들이 하드웨어 제약을 극복하기 위해 사용했던 **창의적인 '기술'**에 있습니다.

예를 들어, 슈퍼 닌텐도(SNES)의 'Mode 7'은 당시로서는 충격적인 기술이었습니다. 2D 콘솔에서 갑자기 바닥이 회전하고 확대/축소되며 원근감을 연출하는 모습은 그야말로 마법 같았죠. 하지만 이 마법은 철저히 수학과 하드웨어 레지스터 제어의 결과물이었습니다.

오늘날 우리는 고성능 GPU와 프로그래머블 셰이더라는 강력한 도구를 가지고 있습니다. 그런데도 여전히 '독특한 스타일', '최적화된 2D 효과', 또는 '향수를 자극하는 시각 연출'을 구현할 때마다 "어떻게 해야 가장 효율적이고 멋지게 만들 수 있을까?" 하는 질문에 부딪힙니다. 단순히 2D 스프라이트를 띄우는 것을 넘어, Mode 7처럼 동적으로 변형되는 2D 지형이나 사용자 인터페이스(UI)를 만들고 싶을 때, 값비싼 3D 엔진 기능까지 동원해야 할까요?

### 해결책: 닌텐도 Mode 7의 지혜를 GLSL 셰이더로 풀어내기

답은 의외로 간단합니다. 과거 닌텐도 개발자들이 그랬던 것처럼, 우리는 **수학**이라는 보편적인 언어와 **GLSL 셰이더**라는 현대의 '마법 지팡이'를 이용해 이 문제를 해결할 수 있습니다. Mode 7의 핵심 원리는 스크린의 각 픽셀에 대응하는 텍스처 좌표를 수학적으로 계산하여 맵을 회전, 확대/축소, 왜곡시키는 것이었습니다. 이를 현대 GLSL 셰이더에서는 `fragment shader`에서 구현할 수 있습니다.

우리가 하려는 것은, 3D 카메라와 3D 모델 없이도, 마치 3D 공간을 움직이는 듯한 2D 평면을 셰이더만으로 그려내는 겁니다.

#### Mode 7의 핵심 원리 (간략히)

Mode 7은 기본적으로 거대한 2D 비트맵 배경 레이어를 마치 평면처럼 펼쳐놓고, 카메라의 위치와 회전, 확대/축소 정보를 기반으로 이 비트맵의 어느 부분을 현재 스크린의 특정 픽셀에 그려야 할지 계산했습니다. 특히, 스크린의 Y축 위치(세로)에 따라 '깊이'를 다르게 적용하여 원근감을 만들어내는 것이 중요합니다. 화면 하단은 가깝게, 상단은 멀리 보이게 하는 식이죠.

#### GLSL로 구현하는 Mode 7 알고리즘

이제 이 아이디어를 GLSL 프래그먼트 셰이더로 옮겨봅시다. 우리는 화면에 그려질 각 픽셀(`gl_FragCoord.xy`)에 대해, 해당 픽셀이 최종적으로 어떤 텍스처 좌표(`uv`)를 가져야 할지 계산할 것입니다.

**핵심 로직 개요:**

1.  **화면 좌표를 카메라 시점 기준으로 변환**: 현재 픽셀의 화면 좌표를 카메라의 중심을 기준으로 한 상대 좌표로 바꿉니다.
2.  **가상 깊이(Perspective) 적용**: 스크린의 Y축 위치에 따라 가상의 '깊이'를 부여합니다. 화면 아래쪽은 가깝게, 위쪽은 멀리 떨어져 있는 것처럼 보이게 합니다. 이 깊이에 따라 X축 스케일을 조절하여 원근감을 연출합니다.
3.  **카메라 회전 적용**: 계산된 '월드 공간'에서의 상대 좌표를 카메라 회전의 역회전을 적용하여 실제 맵 좌표로 변환합니다.
4.  **카메라 이동 적용**: 카메라의 월드 위치를 더하여 최종 맵 상의 절대 좌표를 얻습니다.
5.  **텍스처 UV 매핑**: 계산된 절대 좌표를 텍스처에 매핑될 UV 좌표로 변환합니다.

아래는 Mode 7과 유사한 효과를 구현하기 위한 GLSL 프래그먼트 셰이더 의사 코드입니다.

```glsl
#version 330 core

out vec4 FragColor; // 최종 출력 색상

uniform sampler2D u_texture;    // 맵 텍스처
uniform vec2 u_cameraPos;       // 카메라의 월드 좌표 (X, Y)
uniform float u_cameraRotation; // 카메라 회전 각도 (라디안)
uniform float u_cameraHeight;   // 카메라의 가상 높이 (깊이 연산에 사용)
uniform float u_fov;            // 시야각 (원근감 정도 조절)
uniform vec2 u_resolution;      // 화면 해상도

void main() {
    // 1. 현재 픽셀의 화면 좌표를 -1.0 ~ 1.0 범위로 정규화 (화면 중앙이 0,0)
    // Mode 7은 보통 화면의 특정 Y축을 '지평선'으로 간주합니다.
    // 여기서는 화면 중앙을 기준으로 계산하되, Y축은 깊이와 연관시킵니다.
    vec2 screen_uv_normalized = (gl_FragCoord.xy / u_resolution) * 2.0 - 1.0;

    // 2. 가상 깊이 (Perspective) 적용
    // screen_uv_normalized.y가 낮을수록(화면 하단) '가깝다'고 가정합니다.
    // screen_uv_normalized.y가 높을수록(화면 상단) '멀리 있다'고 가정합니다.
    // 카메라 높이와 시야각을 이용해 스크린 Y 위치를 월드 깊이로 변환합니다.
    float world_depth = u_cameraHeight / (screen_fov_y * (screen_uv_normalized.y * 0.5 + 0.5)); // 대략적인 깊이 연산
    // 좀 더 정교한 계산은 카메라 모델과 투영 행렬을 사용하지만,
    // Mode 7 느낌을 위해 간단한 비례식을 사용합니다.
    // 여기서 (screen_uv_normalized.y * 0.5 + 0.5)는 0.0~1.0 범위로 변환된 Y값.
    // 이 값이 작을수록 (화면 상단) world_depth는 커집니다.

    // 극단적인 값 방지
    world_depth = max(0.001, world_depth);
    world_depth = min(1000.0, world_depth); // 최대 렌더링 거리 제한

    // 3. 월드 공간에서의 상대 X 좌표 계산
    // 화면 X좌표는 깊이에 비례하여 스케일이 달라집니다.
    float world_x_relative = screen_uv_normalized.x * world_depth / u_fov;

    vec2 world_pos_relative = vec2(world_x_relative, world_depth);

    // 4. 카메라 회전 적용 (월드 공간 상대 좌표를 회전)
    float cos_rot = cos(u_cameraRotation);
    float sin_rot = sin(u_cameraRotation);
    mat2 rot_matrix = mat2(
        cos_rot, -sin_rot,
        sin_rot,  cos_rot
    );
    vec2 rotated_world_pos_relative = rot_matrix * world_pos_relative;

    // 5. 카메라 이동 적용 (절대 월드 좌표 계산)
    vec2 absolute_world_pos = u_cameraPos + rotated_world_pos_relative;

    // 6. 텍스처 UV 매핑
    // 맵 텍스처의 크기(예: 100.0x100.0)에 맞춰 UV 스케일을 조정하고,
    // fract() 함수로 텍스처 반복을 구현합니다 (타일링 맵의 경우).
    vec2 final_uv = absolute_world_pos / 100.0; // 맵의 가상 크기에 따라 조정
    final_uv = fract(final_uv); // UV가 0~1 범위를 벗어나면 반복

    FragColor = texture(u_texture, final_uv);

    // 옵션: 깊이에 따른 안개 효과나 색상 조절 등을 추가할 수 있습니다.
    // FragColor = mix(FragColor, vec4(0.0, 0.0, 0.0, 1.0), world_depth / 1000.0);
}
```

**설명**:
이 셰이더는 각 픽셀에 대해 `gl_FragCoord.xy`를 이용하여 화면 상의 위치를 얻고, 이를 카메라 위치, 회전, 높이 및 시야각과 결합하여 텍스처의 어느 지점을 샘플링해야 하는지 계산합니다. 핵심은 `world_depth`와 `world_x_relative` 계산에서 Y축 위치에 따른 원근감과 스케일 변화를 적용하는 것입니다. 이 과정은 마치 3D 공간의 평면을 스크린에 투영하는 과정을 역으로 수행하는 것과 유사합니다.

### 시니어 개발자의 의견: 왜 옛 기술을 다시 봐야 하는가?

어떤 분들은 이렇게 생각할 수도 있습니다. "요즘 3D 엔진은 이 모든 걸 더 쉽게 해주는데, 왜 굳이 복잡하게 셰이더로 옛날 기술을 구현하려고 하죠?" 좋은 질문입니다! 여기에 제 경험담을 섞어 답을 해드리죠.

한번은 모바일 게임에서 독특한 UI 전환 효과를 구현해야 했습니다. 화면 전체가 물결치듯 일그러지며 다음 화면으로 넘어가는 효과였죠. 처음에는 복잡한 3D 메쉬 변형을 생각했지만, 성능 오버헤드가 너무 컸습니다. 문득 Mode 7이 떠올랐습니다. "정사각형 평면의 UV를 조작해서 원근감을 만들 수 있다면, 파형 함수를 UV에 더해서 물결 효과도 만들 수 있지 않을까?"

결과는 대성공이었습니다. 훨씬 적은 연산으로 원하는 시각 효과를 얻었을 뿐만 아니라, 셰이더 내부에서 모든 것이 처리되니 CPU에 부하를 주지 않아 성능 최적화에도 큰 도움이 됐습니다. 이 경험을 통해 저는 깨달았습니다. **제약은 창의력을 낳고, 오래된 기술은 새로운 문제 해결의 열쇠가 될 수 있다**는 것을요.

Mode 7의 수학적 원리를 이해하는 것은 단순히 레트로 게임을 만드는 것을 넘어, 다음과 같은 곳에 응용될 수 있습니다.

*   **독특한 2D UI/UX 애니메이션**: 텍스처 좌표를 직접 조작하여 기존에는 볼 수 없던 다이내믹한 2D UI 효과를 만듭니다.
*   **최적화된 맵 렌더링**: 특정 시점에서만 필요한 복잡한 3D 지형 대신, 셰이더 기반 2D 평면 렌더링으로 성능을 확보합니다.
*   **프로시저럴 텍스처/이펙트**: 셰이더 내에서 수학적 함수를 이용하여 실시간으로 텍스처를 생성하거나 동적인 파티클 효과를 만듭니다.
*   **커스텀 포스트 프로세싱**: 화면 전체에 걸쳐 특정 시각 필터를 적용할 때, Mode 7과 같은 좌표 변환 기법을 응용할 수 있습니다.

### 결론: 과거에서 배우고, 현재를 혁신하다

오늘 우리는 'Shader Math & GLSL'이라는 현대의 강력한 도구를 활용해 'Development Techniques of Retro Nintendo Games'의 정수라 할 수 있는 Mode 7의 마법을 해부하고 재현해봤습니다. 닌텐도 개발자들이 제한된 하드웨어에서 피와 땀으로 일궈낸 수학적 통찰은, 수십 년이 지난 지금도 우리에게 깊은 영감과 실질적인 문제 해결 능력을 제공합니다.

기술은 끊임없이 발전하지만, 그 근간을 이루는 수학적 원리와 창의적인 사고방식은 변하지 않습니다. 옛 기술을 깊이 이해하는 것은 단순히 과거를 답습하는 것이 아니라, 미래의 혁신을 위한 강력한 기반을 다지는 일입니다.
