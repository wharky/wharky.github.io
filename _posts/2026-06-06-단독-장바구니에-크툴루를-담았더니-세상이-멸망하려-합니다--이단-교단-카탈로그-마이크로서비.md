## 🛠️ 눈물겨운 솔루션 아키텍처: "차단막(Rate Limiter)과 영혼 청소기"

현장에 도착해보니 서버실 공기가 이미 물리 법칙을 무시하고 있었습니다. 모니터 선들은 뱀처럼 기어 다니고, 쿨러 팬에서는 "F'gof'nn..." 하는 음산한 고대어 소리가 흘러나왔습니다.

이 장애를 해결하기 위해 제가 즉석에서 설계한 긴급 아키텍처는 다음과 같았습니다.

```
[폭주하는 신도] ──(광클!)──> [API 게이트웨이] ──(트래픽 폭발!)──> [소환 서버 (메모리 누수)]
                                    │
                         [긴급 처방: 디바운스 필터링]
                                    │
                                    ▼
[봉인용 데드 레터 큐(Dead Letter Queue)] ──> [블랙홀로 쓰레기 데이터 폐기]
```

### 1. '디바운스(Debounce)' 보호막 치기
이게 제일 중요했습니다. 신도가 버튼을 1초에 100번 누르더라도, **"가장 마지막에 누른 단 한 번의 요청만 진짜로 인정하겠다"**는 필터를 걸어야 했습니다. 
일상생활로 치면, 벨을 띵동띵동띵동 누르는 장난꾸러기 초등학생의 손목을 턱 잡고 "마지막 한 번만 누른 걸로 친다?" 하는 물리적 제어 장치입니다.

### 2. '데드 레터 큐(Dead Letter Queue)'로 촉수 격리하기
이미 대기열에 쌓여서 출격을 기다리는 41,999마리의 고대신 복제 데이터들은 그냥 지우면 차원에 찌꺼기가 남습니다. 
따라서 처리 불가능한 에러 주문서들을 모아두는 쓰레기통, 즉 '데드 레터 큐'를 임시로 개설하여 그리로 트래픽을 우회시켰습니다. 쉽게 말해 배달 주소를 '태양 한가운데'로 바꿔버린 거죠.

## 💻 장애를 일으킨 범인: 자바스크립트 가상 코드

문제가 된 교단의 `summon-service.js` 코드를 열어보았습니다. 소스 코드를 보자마자 저는 이마를 탁 쳤습니다. 동기식(Sync)과 비동기식(Async)의 혼종이 빚어낸 대참사였습니다.

```javascript
// ❌ 장애를 유발한 원본 코드 (주석을 주목하세요)
async function purchaseAndSummon(userSession, elderGodId) {
  // 1. 장바구니에 담긴 고대신의 정보를 카탈로그에서 긁어옵니다.
  const godInfo = await CatalogDb.find(elderGodId);

  // 2. [문제의 구간] 사용자의 영혼 잔액을 확인합니다.
  // 비동기 처리(await)가 없어서, 잔액이 깎이기도 전에 다음 소환 요청이 통과됩니다!
  if (userSession.soulBalance >= godInfo.cost) {
    
    // 3. 소환 포털 가동 (이것도 비동기인데 처리를 안 해놔서 무한 루프 돌기 딱 좋음)
    PortalEngine.openGate(godInfo.coordinates);
    
    // 4. 결제 완료 처리
    userSession.soulBalance -= godInfo.cost;
    console.log("소환 완료! 영혼이 차감되었습니다.");
  } else {
    throw new Error("영혼 잔액이 부족합니다. 더 많은 기부금을 내세요.");
  }
}
```

이 코드는 쉽게 비유하자면, **통장 잔액이 1만 원밖에 없는데 ATM기 10대에서 동시에 '1만 원 출금' 버튼을 눌렀더니 10만 원이 나와버리는 치명적인 버그**입니다. 고대신 버전으로요.

이를 해결하기 위해 제가 긴급 수정 처방을 내린 소스 코드입니다.

```javascript
//  [심폐소생술] 디바운스 및 뮤텍스 락(Mutex Lock)을 적용한 안전 코드
const mutexLock = new Map(); // 중복 소환을 막기 위한 우주적 자물쇠

async function safeSummon(userSession, elderGodId) {
  // 이미 소환 프로세스가 진행 중인 유저라면 즉시 차단 (서버 터짐 방지)
  if (mutexLock.get(userSession.userId)) {
    console.warn("이미 소환 게이트가 열려 있습니다. 얌전히 기다리십시오, 필멸자여.");
    return;
  }

  // 자물쇠 철컥! (이 유저의 세션을 잠금 처리)
  mutexLock.set(userSession.userId, true);

  try {
    const godInfo = await CatalogDb.find(elderGodId);
    
    // 영혼 잔액 차감과 소환을 트랜잭션(Transaction) 하나로 묶어 안전하게 처리
    await Database.transaction(async (trx) => {
      const currentBalance = await trx.getBalance(userSession.userId);
      
      if (currentBalance >= godInfo.cost) {
        // 실제 게이트 오픈 및 영혼 실시간 차감 (동시성 보장)
        await trx.deductSoul(userSession.userId, godInfo.cost);
        await PortalEngine.openGateSecurely(godInfo.coordinates);
      } else {
        throw new Error("영혼 부족");
      }
    });

  } catch (error) {
    // 에러 발생 시 쓰레기 데이터는 우주의 무덤(Dead Letter Queue)으로 폐기
    DeadLetterQueue.send(elderGodId, error.message);
  } finally {
    // 모든 작업이 끝나면 자물쇠를 풀어줍니다.
    mutexLock.delete(userSession.userId);
  }
}
```
