## 💡 범인은 누구인가? (feat. 마법의 찰떡 비유)

원인을 분석해 보니, 주니어 네크로맨서 개발자가 작성한 **'프론트엔드 코드'**가 문제였습니다. 

종말 교단의 카탈로그 웹사이트는 **React(리액트)**로 구축되어 있었습니다. 여기서 두 가지 치명적인 마법 오작동(버그)이 일어났죠.

### 1. 하이드레이션(Hydration)의 폭주
* **진짜 뜻**: 서버가 미리 만들어둔 뼈대(HTML)에, 브라우저가 동적인 영혼(JavaScript)을 불어넣는 과정입니다.
* **찰떡 비유**: 이건 마치 **'인스턴트 컵라면에 뜨거운 물을 붓는 것'**과 같습니다. 건조된 면발(HTML)에 물을 부어야 탱글탱글한 라면(interactive한 웹페이지)이 되죠.
* **문제점**: 주니어 녀석이 소환 의식 코드에 하이드레이션 설정을 잘못 건드렸습니다. 고대신 '가그나골(Gagna'gol)'의 데이터를 렌더링하는데, 컴퓨터 메모리에 물을 부으라고 했더니 **진짜 심해의 눅눅한 차원 수문(Water Portal)을 열어서 실제 바닷물을 서버실에 부어버린 겁니다.**

### 2. 메모리 누수를 동반한 무한 스크롤(Infinite Scroll)
* **진짜 뜻**: 스크롤을 내릴 때마다 데이터를 계속 불러오는데, 이전 데이터를 메모리에서 안 지우고 계속 쌓아두는 버그입니다.
* **찰떡 비유**: 영수증을 자르지 않고 **지구 한 바퀴 반 바퀴만큼 길게 뽑아내는 영수증 프린터**와 같습니다. 결국 프린터가 종이 무게를 못 이기고 터지겠죠?
* **문제점**: 신도들이 제물 카탈로그를 보려고 스크롤을 무한으로 내리자, 램(RAM) 메모리에 수만 마리의 '제물용 염소' 데이터가 중복으로 쌓였습니다. 과부하가 걸린 서버가 비명을 지르며 차원의 틈새를 열어버린 것입니다.

## 🛠️ 눈물겨운 솔루션 아키텍처

질퍽거리는 서버실에 장화를 신고 들어가, 촉수들을 피해 가며 긴급 복구 작업을 시작했습니다.

```
[신도들의 웹 브라우저]
       │
       ▼ (무한 스크롤 연타!)
[디바운스 필터 (Debounce)] ───▶ "야, 0.5초 동안 한 번만 요청해!" (진정제 투여)
       │
       ▼ (통과된 요청)
[가상 스크롤러 (Virtual Scroll)] ───▶ "화면에 보이는 제물 5개만 그리고 나머지는 가상 차원에 짱박아!"
       │
       ▼ (안전한 데이터)
[가비지 컬렉터 (Garbage Collector)] ───▶ "다 본 염소 데이터는 즉시 소각한다."
```

1. **가상 스크롤(Virtual Scroll) 도입**: 
   만 개의 제물 이미지를 한 번에 다 그리지 않고, 오직 신도의 모니터 화면에 보이는 '5개'만 렌더링하게 했습니다. 화면 밖으로 벗어난 염소들은 즉시 메모리에서 증발시켰습니다.
2. **디바운싱(Debouncing) 마법 적용**: 
   신도들이 마우스 휠을 미친 듯이 굴려도, 0.5초 동안의 스크롤 입력 중 가장 마지막 입력 딱 한 번만 서버에 요청을 보내도록 기도를 제한했습니다.
3. **가비지 컬렉터 강제 구동**: 
   주기적으로 메모리를 청소하는 은빛 화살 마법을 서버 엔진에 주입해, 소환 의식이 실패한 찌꺼기 세션들을 즉각 소멸시켰습니다.

//  우아하고 안전한 시니어의 해결 코드
import { throttle } from 'elven-magic-library';

// 화면에 보이는 구역만 다루는 가상 돔 매니저

// 스크롤이 발생할 때 0.3초의 쿨타임(디바운스)을 적용함

    summonEmergencyPaladin(); // 예외 처리: 긴급 성기사 소환

다행히 아침 9시 데일리 스크럼(조회) 전까지 동기화를 마쳐서, 교주님께 조인트 까이는 건 면했네요.

"김 팀장, 이번 분기 종말 KPI 달성률이 왜 이 모양인가?" 하고 교주님이 물으시길래, **"고대신 포털 API의 레이턴시(지연 시간)가 길어서 서킷 브레이커가 걸렸습니다"**라고 대충 그럴싸한 IT 용어로 둘러댔더니 고개를 끄덕이시더군요. 역시 문과 출신 교주들은 영단어만 섞어 쓰면 다 믿습니다.

다음 이직 시즌에는 꼭 엘프들이 운영하는 평화로운 포션 쇼핑몰로 도망쳐야겠습니다. 거기는 버그가 나봤자 포션 색깔이 핑크색으로 변하는 정도지, 최소한 서버실에 바닷물이 차오르진 않으니까요.

오늘의 교훈: **무한 스크롤 구현할 때는 반드시 가상 돔(Virtual DOM)과 디바운스를 쓰자. 안 그러면 당신의 서버실이 아틀란티스가 될 것이다.**
